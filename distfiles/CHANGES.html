Detroubulator $version$ - CHANGES

This document contains an overview of the new features introduced with each new release of Detroubulator. For more in-depth information, please refer to the documentation library.

Version 1.1

New assertions

Two new assertions have been introduced. The validation assertion validates the mapping program's output against a schema. If the output is valid, the validation assertion passes. If it is not, the validation assertion fails. The external assertion is a Java class that implements the org.detroubulator.core.Assertion interface. This lets you write assertions that can examine the mapping program's output down to the byte level. This is useful in two cases:

  • When you need to examine the mapping program's output in more detail than is possible with XPath
  • When the mapping program's output is not XML

Support for all XI mapping programs

Detroubulator 1.0 shipped with support for Java, XSLT and message mapping programs. Version 1.1 introduces support for ABAP mapping programs, ABAP XSLT mapping programs and interface mappings.

Shared assertions

Shared assertions are named groups of assertions that can be referenced from multiple test cases. Shared assertions are useful in cases where the same assertions occur in several test cases. Rather than copying the assertions between test cases, create an assertion group and reference it from the test cases.

Expected values in content assertions

In Detroubulator version 1.0, the content assertion contained an XPath expression, which was expected to evaluate to Boolean true. The version 1.1 content assertions lets you explicitly specify the expected value in the new expect attribute. This makes for much improved error messages when the expected and the actual value are different.

Miscellaneous

  1. Detroubulator is now a 100% command line utility
  2. The design of the HTML output report has been updated and no longer requires images loaded from the Detroubulator website
